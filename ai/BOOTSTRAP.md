# AI Knowledge System — Bootstrap

> **Single source of truth** for all AI agents working on this React project. Agent-agnostic. Platform adapters (`CLAUDE.md`, `.github/copilot-instructions.md`) reference this directory.

## Stack

React 19 | TypeScript ~5.9 | Vite 8 | TanStack Router (file-based) | TanStack Query 5 | Zustand 5
React Hook Form 7 + Zod 4 | Tailwind v4 + shadcn/ui (Radix + CVA) | Axios | Vitest + RTL | Playwright | Storybook 10
Architecture: **Feature-Sliced Design** enforced by `eslint-plugin-boundaries`

## Architecture

```
src/
├── main.tsx              # Vite entry — mounts <App /> + providers
├── routeTree.gen.ts      # AUTO-GENERATED — never edit by hand
├── routes/               # TanStack Router file-based routes
│   └── __root.tsx        # Root layout (devtools mounted here)
├── app/                  # App-level providers, error boundary, query client
├── features/<name>/      # Self-contained feature modules
│   ├── api/              # Axios calls + queryOptions / mutationFns
│   ├── components/       # Smart components (use stores/queries)
│   ├── hooks/            # Feature-local hooks
│   ├── stores/           # Zustand stores (if any)
│   ├── types/            # Feature types & Zod schemas
│   ├── utils/            # Feature-local helpers
│   └── assets/           # Feature-local assets
├── components/           # Shared presentational — NO feature imports
├── ui/                   # shadcn/ui primitives (Button, Link, Spinner…)
├── hooks/                # Cross-feature hooks
├── lib/                  # Shared singletons: api.ts, react-query.ts
├── config/               # env.ts (Zod-validated env)
├── types/                # Cross-feature TS types
├── utils/                # Pure helpers (cn, formatters…)
├── testing/              # Test setup, helpers, MSW handlers
├── assets/               # Global assets
└── index.css             # Tailwind v4 + theme tokens
```

### Boundary rules (ESLint-enforced)

- A feature MUST NOT import from another feature.
- A feature MUST NOT import from `app/`.
- `components/`, `hooks/`, `lib/`, `types/`, `utils/` MUST NOT import from `features/` or `app/`.
- All cross-directory imports MUST use `@/` alias — no `../../`.
- No import cycles.

## Critical Rules (Tier 0 — always enforced)

| #   | Rule                         | Details                                                                                        |
| --- | ---------------------------- | ---------------------------------------------------------------------------------------------- |
| 1   | NO `console.log`             | Only `console.warn` / `console.error`. Use a logger if persistence is needed.                  |
| 2   | NO `any`                     | `@typescript-eslint/no-explicit-any: error`. Use `unknown` + narrowing.                        |
| 3   | `@/` alias only              | No `../../` imports. Enforced by `no-restricted-imports`.                                      |
| 4   | Files & folders = kebab-case | `check-file/filename-naming-convention` and `folder-naming-convention`.                        |
| 5   | Feature isolation            | Features cannot import from other features (`eslint-plugin-boundaries`).                       |
| 6   | Shared layers stay pure      | `components/`, `hooks/`, `lib/`, `types/`, `utils/` cannot import features or app.             |
| 7   | Boolean naming               | `is*` / `has*` / `should*` / `can*` / `did*` / `will*` / `are*` (variables and parameters).    |
| 8   | Components ≤150 lines        | `max-lines: warn` in `components/**/*.tsx`.                                                    |
| 9   | No nested ternary            | `no-nested-ternary: error`.                                                                    |
| 10  | Strict equality              | `eqeqeq: error` (null comparison allowed).                                                     |
| 11  | Import order                 | Builtin → external → internal → parent/sibling → index → object, alphabetized.                 |
| 12  | No cycles                    | `import/no-cycle: error`.                                                                      |
| 13  | Blank line before return     | `padding-line-between-statements: error`.                                                      |
| 14  | `routeTree.gen.ts`           | NEVER edit — auto-generated by TanStack Router plugin.                                         |
| 15  | Conventional commits         | `commitlint` enforces `feat\|fix\|docs\|refactor\|test\|chore\|deps\|perf\|revert\|ci\|build`. |

### Loading protocol

1. **Always loaded**: This file (`BOOTSTRAP.md`) — keep <200 lines.
2. **On trigger**: Check `manifest.json` → match task keywords → `triggers` field → load matching skill.
3. **On demand**: Full protocols, agent personas — load only when explicitly needed.

## Engine (Complex Tasks)

For tasks involving >3 files or parallel operations, check `ai/engine/registry.json` for matching scenarios. Scenarios compose 11 atomic nodes (`discover`, `extract`, `spawn`, `gate`, `verify`, …) into DAG pipelines. See `ai/README.md` §4 for full documentation.
